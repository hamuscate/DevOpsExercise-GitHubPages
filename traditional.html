A customer who uses your SaaS service generates 20 million events per day.

Each event should be forwarded to a syslog server but customer is reporting that for periods of time they are seeing that forwarding completely stops - about 2-3 times every 7 days.

When forwarding stops the outage can last hours.

Assumptions

The problem is too complex to solve from logs alone.
When the problem occurs, the team can restart the forwarding service which resolves the issue until the problem occurs again

Constraints

It takes 3 days to work through the ops process to create a patch to add diagnostics.
New software updates are only deployed once every month after 1 week of validation.

Tips

Solution description should focus on outlining the process - don't worry about specific tools too much. Key concepts are fast feedback loops, automation, and runtime monitoring/telemetry.
